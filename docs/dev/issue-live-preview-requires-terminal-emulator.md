# Issue: ライブプレビューに端末エミュレータが必要

## 重大度: MEDIUM

## 問題

`claude-switch.sh` のプレビューウィンドウで tmux ペインをリアルタイム表示したい。
現在の実装 (`claude-live-preview.sh`) は 50ms ポーリングで `capture-pane -e` を呼び出し、
変化があった場合のみ fzf に出力する。

fzf はプレビューの出力が届くたびに画面を全面クリア＆再描画するため、
変化があるたびに一瞬フラッシュし、静止状態と変わりがない見た目になる。
真の「ストリーミング描画」にはならない。

## 試みた別アプローチ

`pipe-pane -o` でペイン出力をファイルにリダイレクト → `tail -n +1 -f` でストリーミング。
しかし `pipe-pane` が吐き出す生の PTY バイト列には
`\r\r\n`, `\033[?2026l` などカーソル制御シーケンスが含まれており、
fzf プレビューのような非 PTY 環境では正しく解釈されず表示が壊れる。

## 根本的な制約

tmux ペインを別の TUI ウィジェット内に「ネイティブに埋め込む」には、
VT100/ANSI 互換の端末エミュレータがそのウィジェット内で動作する必要がある。
これは純粋な shell スクリプトでは実現できない。

## 推奨解決策

Go で CLI ツールを書き直す際に以下を利用する:

- **tcell-term**: Go の tcell UI フレームワーク向け端末エミュレータウィジェット
  - https://github.com/nicholasgasior/tcell-term
- **go-vt10x**: VT10x 端末ステートマシン（描画には別途 tcell が必要）
  - https://github.com/hinshun/go-vt10x

これらを使えば tmux ペインの出力を pipe-pane でキャプチャし、
内蔵エミュレータで解釈して TUI 内に正しくレンダリングできる。
lazygit の差分ビューや gocui のパネルと同等のリアルタイム表示が可能になる。

## 現状の暫定対応

`claude-live-preview.sh` の 50ms capture-pane ポーリングを維持する。
完全な静止表示よりは若干ましだが、真のリアルタイムではない。
